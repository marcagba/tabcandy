About

One of the slides in the recently presented Firefox 4 roadmap had a mainly unexplained grey bullet point on TabCandy.

TabCandy is a feature we've been exploring for Firefox on how to handle the modern web workflow by being able to name and group tabs into work spaces. Think Expos√© meets Spaces done right. It's an attempt to solve many of the problems we've seen emerge over the last number of years around multitasking and information proliferation. The reason why TabCandy was listed in grey in the product plan is that there are still some some implementation challenges to be solved and a couple user experience questions to be answered for us to be able to commit to TabCandy being in Firefox 4.

Code Intro.

Getting Started

TabCandy itself is a webpage that runs in chrome space. It currently lives inside of a standard browser tab which we hide from the tabstrip (one of the things we know we will probably need to change). This page can be found in content/candies/revision-a/index.html. All of the user interface code lives in revision-a/js with ui.js being the most high-level.

There are a whole bunch of visualizations in content/candies but you can ignore all of them except revision-a. We'll probably be removing all but revision-a for launch.

Most of the lower-level tab watching, utilities, and dark magic is done in content/js/core with tabs.js being the most low-level.
